# 前端交互页面设计方案

## 设计目标

- **简单易用**：操作直观，查看方便
- **快速开发**：最小化MVP，1-2周完成
- **美观现代**：现代化UI设计
- **功能完整**：覆盖核心使用场景

## 功能需求

### 核心功能

1. **数据采集控制**
   - 输入关键词
   - 选择数据源（App Store、Product Hunt等）
   - 设置采集数量
   - 启动/停止采集
   - 查看采集进度

2. **机会查看**
   - 机会列表（表格形式）
   - 排序和筛选
   - 搜索功能
   - 分页显示
   - 详情查看

3. **数据分析**
   - 查看评分详情（各维度分数）
   - 数据可视化（图表）
   - 统计信息

4. **配置管理**
   - 修改评分权重
   - 调整筛选阈值
   - 保存配置

### 扩展功能（后续）

5. **历史记录**
   - 查看历史采集记录
   - 对比不同时间的机会

6. **导出功能**
   - 导出CSV
   - 导出报告

## 技术栈选择

### 方案1：Flask + Tailwind CSS + Flowbite（推荐⭐）

**优点**：
- ✅ **现代化设计**：比Bootstrap更美观专业
- ✅ **高度可定制**：实用工具类，灵活
- ✅ **Flask集成**：有官方Flask集成文档
- ✅ **组件丰富**：Flowbite提供200+组件
- ✅ **文件体积小**：按需加载
- ✅ **响应式设计**：移动端友好
- ✅ **暗色模式**：内置支持

**技术栈**：
- 后端：Flask（Python）
- 前端：Tailwind CSS + Flowbite
- 图表：Chart.js
- 表格：DataTables

**适用**：追求专业美观的现代化UI

**开发时间**：1.5-2周

### 方案2：Flask + Bootstrap 5（备选）

**优点**：
- ✅ 开发快速（1-2周）
- ✅ 组件最丰富
- ✅ 文档最完善
- ✅ 学习成本最低

**技术栈**：
- 后端：Flask（Python）
- 前端：Bootstrap 5 + jQuery
- 图表：Chart.js
- 表格：DataTables

**适用**：追求最快开发速度

### 方案2：Flask + Vue.js

**优点**：
- 现代化前端框架
- 交互体验好
- 可扩展性强

**缺点**：
- 开发时间稍长（2-3周）
- 需要学习Vue.js

**适用**：如果需要更好的交互体验

### 方案3：Next.js + Python API

**优点**：
- 前后端完全分离
- 现代化技术栈
- SEO友好

**缺点**：
- 开发时间较长（3-4周）
- 需要维护两个项目

**适用**：长期项目，需要更好的架构

## 推荐方案：Flask + Tailwind CSS + Flowbite

### 理由

1. **更专业美观**：现代化设计，比Bootstrap更精致
2. **Flask友好**：有官方Flask集成文档和示例
3. **组件丰富**：Flowbite提供200+现成组件
4. **高度可定制**：实用工具类，灵活设计
5. **技术现代**：符合2024年设计趋势
6. **部署简单**：单进程部署

### 备选方案：Flask + Bootstrap 5

如果追求**最快开发速度**，Bootstrap 5仍然是好选择：
- 文档最完善
- 组件最丰富
- 学习成本最低

**详细对比请查看**：[框架对比文档](FRAMEWORK_COMPARISON.md)

## UI/UX设计

### 设计风格

- **现代化**：使用Bootstrap 5的现代设计
- **简洁**：界面简洁，信息清晰
- **响应式**：支持不同屏幕尺寸
- **易用**：操作直观，反馈及时

### 颜色方案

- **主色**：蓝色系（专业、可信）
- **成功**：绿色（采集成功、机会发现）
- **警告**：黄色（进行中、需要注意）
- **错误**：红色（错误、失败）

### 交互设计

1. **实时反馈**
   - 采集进度实时更新
   - 操作按钮状态反馈
   - 加载动画

2. **数据可视化**
   - 评分详情用进度条显示
   - 统计信息用卡片展示
   - 趋势用图表展示

3. **快捷操作**
   - 常用功能一键访问
   - 键盘快捷键支持
   - 批量操作

## 技术实现

### 后端API设计

```python
# Flask API路由
/api/v1/scrape/start      # 启动采集
/api/v1/scrape/status     # 获取采集状态
/api/v1/scrape/stop       # 停止采集
/api/v1/opportunities     # 获取机会列表
/api/v1/opportunities/<id> # 获取机会详情
/api/v1/config            # 获取/更新配置
/api/v1/stats             # 获取统计信息
```

### 前端组件

1. **采集控制组件**
   - 关键词输入
   - 参数设置
   - 进度显示

2. **机会列表组件**
   - 数据表格
   - 筛选器
   - 分页器

3. **详情展示组件**
   - 信息卡片
   - 评分可视化
   - 图表展示

4. **配置管理组件**
   - 滑块控件
