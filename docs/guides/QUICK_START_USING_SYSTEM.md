# 快速开始使用系统

## 🎯 目标
使用系统发现真实的商业机会，然后开始MVP开发。

## 📋 第一步：安装依赖

如果还没有安装 `openpyxl`（用于Excel导出），请运行：

```bash
pip install openpyxl>=3.1.0
```

或者安装所有依赖：

```bash
pip install -r requirements.txt
```

## 🚀 第二步：启动系统

```bash
cd frontend/web
python app.py
```

然后在浏览器打开：`http://localhost:5000`

## 📊 第三步：采集数据

### 3.1 采集App Store数据

1. **在首页输入关键词**
   - 例如：`fitness`, `meditation`, `productivity`
   - 可以输入多个关键词，用逗号分隔

2. **点击"开始采集"**
   - 系统会自动搜索App Store
   - 实时显示采集进度
   - 采集完成后会显示发现的机会数量

3. **查看机会列表**
   - 点击"查看机会列表"
   - 按机会分数排序，找到Top机会
   - 点击"查看详情"深入了解

### 3.2 采集搜索趋势数据

1. **进入趋势页面**
   - 点击导航栏的"搜索趋势"

2. **添加关键词**
   - 输入关键词，按Enter添加
   - 可以添加多个关键词

3. **开始采集**
   - 选择时间范围（建议：过去12个月）
   - 点击"开始采集"
   - 等待采集完成

4. **分析趋势**
   - 查看热门关键词（增长率高的）
   - 点击"分析"查看详细趋势分析
   - 查看趋势图表

## 📥 第四步：导出数据

### 4.1 导出机会列表

1. **在机会列表页面**
   - 可以使用筛选功能找到感兴趣的机会
   - 点击右上角"📥 导出数据"
   - 选择格式（CSV 或 Excel）

2. **使用导出的数据**
   - 在Excel中进行深度分析
   - 制作数据透视表
   - 生成图表
   - 分享给团队成员

### 4.2 导出趋势数据

1. **在趋势页面**
   - 选择要导出的关键词（可选）
   - 点击"📥 导出趋势数据"
   - 文件会自动下载

2. **使用导出的数据**
   - 在Excel中分析趋势
   - 使用Python进行时间序列分析
   - 制作趋势报告

## 🔍 第五步：分析机会

### 5.1 评估标准

参考 `docs/reference/inspiration/INSPIRATION_数据挖掘和分析框架.md` 中的评估维度：

1. **市场规模**（30%）
   - 评论数、下载量、搜索量
   - 使用Google Trends验证

2. **竞争程度**（25%）
   - 竞品数量和质量
   - 市场饱和度

3. **用户满意度**（20%）
   - 评分和评论
   - 用户痛点和需求

4. **增长趋势**（15%）
   - 搜索趋势
   - 市场增长潜力

5. **变现潜力**（10%）
   - 商业模式
   - 付费意愿

### 5.2 快速评估方法

1. **查看机会分数**
   - 分数越高，机会越大
   - 建议关注分数 > 0.7 的机会

2. **查看详情**
   - 应用描述（可以翻译成中文）
   - 评分和评论数
   - 截图和功能

3. **验证趋势**
   - 使用趋势功能验证市场规模
   - 查看搜索热度变化

4. **竞品分析**
   - 下载Top竞品体验
   - 分析功能和差异化点

## 🎯 第六步：选择机会

### 选择标准

1. **至少3个维度为"高"**
   - 市场规模大
   - 竞争不激烈
   - 用户满意度高

2. **技术可行性高**
   - 1-2周可完成MVP
   - 技术栈熟悉

3. **有明确的差异化点**
   - 功能差异化
   - 体验差异化
   - 定位差异化

## 🚀 第七步：开始MVP开发

### MVP开发原则

1. **只做核心功能**（不超过3个）
2. **1-2周完成**
3. **快速上线验证**

### 验证渠道

1. **Product Hunt**（海外）
2. **Reddit相关社区**
3. **酷安**（国内）
4. **朋友圈/微信群**

### 验证指标

1. **1周内下载量 > 100？**
2. **次日留存 > 30%？**
3. **有正面用户反馈？**
4. **有用户愿意付费？**

## 💡 关键建议

1. **快速行动**：不要过度分析，快速验证更重要
2. **小步快跑**：1-2周完成MVP，快速上线
3. **数据驱动**：用数据做决策，而不是猜测
4. **快速放弃**：验证失败立即放弃，尝试下一个
5. **持续优化**：根据实际使用情况优化系统
