# 个人创业者核心能力实践练习手册

## 🎯 目标

通过系统性的实践练习，全面提升个人创业者的核心能力：
- 商业思维（基础层）
- 需求能力（核心层）
- 产品能力（执行层）
- 验证能力（关键层）
- 执行能力（保障层）
- 学习能力（持续层）
- 心理素质（支撑层）

## 📅 12周完整实践计划

> **注意**：这是完整的12周计划，你可以根据自己的情况调整节奏。建议至少完成前4周的基础练习。

### 第1周：市场分析基础

#### 练习1：选择一个市场并定义边界（30分钟）

**任务**：
1. 列出5个你感兴趣的市场
2. 选择一个市场
3. 定义市场边界：
   - 这个市场包括什么？
   - 不包括什么？
   - 如何划分细分市场？

**输出**：
- 市场定义文档（1页）

**示例**：
```
市场：在线教育
边界：
- 包括：在线课程、在线培训、在线学习工具
- 不包括：线下教育、传统教育机构
- 细分：K12教育、职业教育、技能培训、语言学习
```

#### 练习2：估算市场规模（1小时）

**任务**：
1. 研究选定市场的规模
2. 估算：
   - TAM（总市场规模）
   - SAM（可服务市场规模）
   - SOM（可获得市场规模）
3. 分析市场增长：
   - 历史增长率
   - 未来增长预测
   - 增长驱动因素

**输出**：
- 市场规模分析（1页）
- 数据来源列表

**工具**：
- Google Trends
- Statista
- 行业报告
- 新闻媒体

#### 练习3：分析竞争格局（1小时）

**任务**：
1. 识别Top 5竞争者
2. 分析：
   - 市场份额
   - 产品特点
   - 差异化点
3. 评估：
   - 竞争激烈程度
   - 新进入者机会

**输出**：
- 竞争格局分析（1页）

**工具**：
- App Store / Google Play
- Product Hunt
- 竞品官网

#### 练习4：分析市场趋势（1小时）

**任务**：
1. 识别3个市场趋势
2. 分析每个趋势：
   - 趋势是什么？
   - 为什么会出现？
   - 会如何影响市场？
   - 会带来什么新机会？

**输出**：
- 市场趋势分析（1页）

**工具**：
- Google Trends
- Twitter / Reddit
- 行业报告

#### 练习5：完成市场分析画布（2小时）

**任务**：
1. 用市场分析画布框架
2. 完成完整的市场分析
3. 形成市场分析报告

**输出**：
- 市场分析报告（2-3页）

**框架**：
```
1. 市场定义
2. 市场规模
3. 市场结构
4. 市场趋势
5. 市场机会
```

### 第2周：需求挖掘基础

#### 练习6：观察用户行为（2小时）

**任务**：
1. 选择一个场景（使用某个App、完成某个任务）
2. 观察3-5个用户的行为
3. 记录：
   - 用户在做什么？
   - 遇到了什么问题？
   - 如何解决的？
   - 有什么痛点？

**输出**：
- 用户行为观察报告（1-2页）

**方法**：
- 直接观察
- 用户测试
- 数据分析
- 社区观察

#### 练习7：进行用户访谈（3小时）

**任务**：
1. 选择一个主题
2. 访谈3-5个用户
3. 问以下问题：
   - 你最痛苦的事情是什么？
   - 你如何解决这个问题？
   - 现有解决方案有什么不足？
   - 你希望有什么工具？
   - 你愿意为这个工具付费吗？

**输出**：
- 用户访谈报告（2-3页）
- 访谈录音/笔记

**访谈技巧**：
- 问"为什么"，而不是"是什么"
- 听用户的故事
- 关注痛点
- 问开放性问题

#### 练习8：深度体验竞品（1周）

**任务**：
1. 选择一个领域
2. 选择3-5个竞品
3. 深度使用1-2周
4. 记录：
   - 每个产品的优点
   - 每个产品的缺点
   - 你的痛点
   - 改进机会

**输出**：
- 竞品体验报告（2-3页）

#### 练习9：社区需求挖掘（1周）

**任务**：
1. 选择一个领域
2. 在相关社区观察1周
3. 收集：
   - 10个用户抱怨
   - 10个需求讨论
   - 5个高频痛点
   - 3个改进机会

**输出**：
- 社区需求挖掘报告（2-3页）

**社区**：
- Reddit
- Twitter
- 论坛
- 微信群

#### 练习10：完成需求挖掘画布（2小时）

**任务**：
1. 用需求挖掘画布框架
2. 完成完整的需求挖掘
3. 形成需求挖掘报告

**输出**：
- 需求挖掘报告（2-3页）

**框架**：
```
1. 目标用户
2. 痛点识别
3. 需求分析
4. 解决方案
5. 需求验证
```

### 第3周：需求分析基础

#### 练习11：需求分类（1小时）

**任务**：
1. 收集10个需求（从访谈、观察、体验中）
2. 按以下维度分类：
   - 需求类型（功能/体验/情感）
   - 需求优先级（必须/期望/惊喜）
   - 需求来源（显性/隐性/潜在）

**输出**：
- 需求分类表（1页）

#### 练习12：需求评估（1小时）

**任务**：
1. 选择5个需求
2. 按以下维度评估：
   - 需求频率
   - 需求强度
   - 需求价值
3. 排序：
   - 按频率排序
   - 按强度排序
   - 按价值排序
   - 综合排序

**输出**：
- 需求评估表（1页）

**评估框架**：
```
高频率 + 高强度 = 核心需求
高频率 + 低强度 = 基础需求
低频率 + 高强度 = 特殊需求
低频率 + 低强度 = 边缘需求
```

#### 练习13：需求验证（2小时）

**任务**：
1. 选择一个需求
2. 用以下方法验证：
   - 用户访谈验证
   - 快速验证（最小版本）
   - 数据验证（用系统）
3. 回答：
   - 需求是否真实？
   - 需求是否普遍？
   - 需求是否可持续？

**输出**：
- 需求验证报告（1-2页）

#### 练习14：完成需求分析画布（2小时）

**任务**：
1. 用需求分析画布框架
2. 完成完整的需求分析
3. 形成需求分析报告

**输出**：
- 需求分析报告（1-2页）

**框架**：
```
1. 需求描述
2. 需求分类
3. 需求评估
4. 需求验证
5. 需求决策
```

### 第4周：综合应用

#### 练习15：完整项目（1周）

**任务**：
完成一个完整的市场分析 → 需求挖掘 → 需求分析 → 需求验证流程

**步骤**：

**第1天：市场分析**
1. 选择一个市场
2. 完成市场分析画布
3. 形成市场分析报告

**第2-3天：需求挖掘**
1. 在选定市场中，选择一个用户群体
2. 用观察、访谈、体验、社区挖掘方法
3. 完成需求挖掘画布
4. 形成需求挖掘报告

**第4天：需求分析**
1. 从挖掘的需求中，选择5个需求
2. 完成需求分析画布
3. 形成需求分析报告
4. 选择1个最有价值的需求

**第5-7天：需求验证**
1. 用系统验证需求（市场规模、竞争情况）
2. 做用户访谈验证
3. 做一个最小版本（MVP）
4. 收集反馈，决定是否继续

**输出**：
- 市场分析报告（2-3页）
- 需求挖掘报告（2-3页）
- 需求分析报告（1-2页）
- 需求验证报告（1-2页）
- MVP（最小版本）

## 💡 练习建议

1. **循序渐进**：从简单到复杂，逐步提升
2. **实践为主**：理论是基础，实践是关键
3. **记录过程**：记录思考过程，形成自己的方法论
4. **持续迭代**：不断优化，持续提升
5. **寻求反馈**：找有经验的人给反馈
