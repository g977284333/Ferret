# 用户使用指南

## 快速开始

### 1. 启动应用

```bash
cd frontend/web
python app.py
```

应用将在 http://localhost:5000 启动

### 2. 访问应用

在浏览器中打开：http://localhost:5000

## 功能使用

### 首页

**功能**：
- 快速开始数据采集
- 查看统计信息
- 浏览最近发现的机会

**操作步骤**：
1. 在"快速开始"区域输入关键词（如：productivity）
2. 按Enter或点击"+ 添加"按钮添加关键词
3. 选择数据源（App Store或Product Hunt）
4. 设置每个关键词的采集数量
5. 点击"开始采集"按钮

**统计信息**：
- 总机会：数据库中所有机会的总数
- 今日采集：今天采集的App数量
- 进行中：当前正在运行的采集任务数

### 数据采集页面

**功能**：
- 管理采集任务
- 查看实时进度
- 查看采集结果

**操作步骤**：
1. 添加关键词（可添加多个）
2. 选择数据源和采集数量
3. 点击"开始采集"
4. 查看实时进度
5. 可以暂停或停止采集

**进度显示**：
- 总体进度：所有关键词的总体采集进度
- 当前关键词进度：当前正在采集的关键词进度
- 预计剩余时间：根据当前速度估算的剩余时间

### 机会列表页面

**功能**：
- 查看所有发现的机会
- 搜索和筛选机会
- 排序和分页
- 导出数据

**操作步骤**：
1. 使用搜索框搜索App名称
2. 使用筛选器筛选（分类、分数范围）
3. 选择排序方式（按分数、评分、评论数）
4. 点击"应用筛选"应用筛选条件
5. 点击"查看详情"查看详细信息
6. 点击"导出CSV"导出数据

**筛选功能**：
- 搜索：按App名称搜索
- 分类：按App分类筛选
- 分数范围：设置机会分数的范围
- 排序：按不同字段排序

### 机会详情页面

**功能**：
- 查看完整的机会信息
- 查看评分详情
- 导出数据

**信息展示**：
- 基本信息：名称、分类、评分、评论数、价格
- 机会分数：总体机会评分
- 评分详情：各维度的详细分数
  - 市场规模（30%）
  - 竞争程度（25%）
  - 用户满意度（20%）
  - 增长趋势（15%）
  - 变现潜力（10%）

**操作**：
- 点击"打开App Store"查看App详情
- 点击"导出数据"导出当前机会的数据
- 点击"返回列表"返回机会列表

### 配置页面

**功能**：
- 调整评分权重
- 设置筛选阈值
- 配置数据源

**操作步骤**：

1. **调整评分权重**：
   - 使用滑块调整各维度的权重
   - 确保权重总和为100%
   - 系统会实时显示权重总和

2. **设置筛选阈值**：
   - 最低机会分数：设置最低分数要求（0.0-1.0）
   - 最少评论数：设置最少评论数要求
   - 最多竞品数：设置最多竞品数量

3. **配置数据源**：
   - 启用/禁用不同的数据源
   - App Store：主要数据源
   - Product Hunt：产品发现平台
   - Google Play：Android应用（待实现）

4. **保存配置**：
   - 点击"保存配置"保存更改
   - 点击"重置"恢复到上次保存的状态
   - 点击"测试配置"测试配置效果

## 常见问题

### 1. 如何开始采集数据？

在首页的"快速开始"区域：
1. 输入关键词（如：productivity, task management）
2. 选择数据源
3. 设置采集数量
4. 点击"开始采集"

### 2. 采集需要多长时间？

取决于：
- 关键词数量
- 每个关键词的采集数量
- 网络速度

通常每个App需要1-3秒，20个App大约需要20-60秒。

### 3. 如何查看采集进度？

采集开始后，会自动跳转到数据采集页面，可以看到：
- 总体进度条
- 当前关键词进度
- 预计剩余时间

### 4. 如何筛选机会？

在机会列表页面：
1. 使用搜索框搜索App名称
2. 选择分类筛选
3. 设置分数范围
4. 点击"应用筛选"

### 5. 如何导出数据？

在机会列表页面：
- 点击"导出CSV"按钮
- 会下载包含所有筛选结果的CSV文件

在机会详情页面：
- 点击"导出数据"按钮
- 会下载当前机会的JSON数据

### 6. 如何调整评分标准？

在配置页面：
1. 使用滑块调整各维度权重
2. 确保权重总和为100%
3. 调整筛选阈值
4. 点击"保存配置"

### 7. 配置保存后何时生效？

配置保存后立即生效，新的采集任务会使用新配置。

## 最佳实践

### 1. 关键词选择
- 使用英文关键词效果更好
- 选择具体的关键词（如：habit tracker而不是tracker）
- 可以添加多个相关关键词

### 2. 采集数量
- 建议每个关键词采集20-50个App
- 太多会耗时较长
- 太少可能遗漏机会

### 3. 筛选机会
- 先使用分类筛选缩小范围
- 再使用分数范围筛选高质量机会
- 最后使用搜索精确查找

### 4. 评分权重调整
- 根据你的关注点调整权重
- 如果关注市场规模，增加市场规模权重
- 如果关注竞争程度，增加竞争程度权重
- 确保权重总和为100%

## 快捷键

- `Enter`：在关键词输入框中按Enter快速添加关键词
- `Ctrl+F`：在机会列表页面快速聚焦搜索框
