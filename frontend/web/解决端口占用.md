# 解决端口5000被占用问题

## 问题
```
Address already in use
Port 5000 is in use by another program.
```

## 解决方案

### 方案1：禁用AirPlay Receiver（推荐）

在macOS上，端口5000通常被AirPlay Receiver占用。

**步骤**：
1. 打开"系统设置"（System Settings）
2. 搜索"AirPlay接收器"（AirPlay Receiver）
3. 关闭"AirPlay接收器"

### 方案2：使用其他端口

我已经修改了代码，如果5000端口被占用，会自动使用5001端口。

启动服务器后，如果看到：
```
⚠️  端口 5000 被占用，使用端口 5001
访问地址: http://localhost:5001
```

那么就在浏览器访问：**http://localhost:5001**

### 方案3：手动停止占用端口的进程

```bash
# 查找占用5000端口的进程
lsof -ti:5000

# 停止进程（替换PID为实际的进程ID）
kill -9 <PID>
```

### 方案4：手动指定端口

如果想使用其他端口，可以修改 `app.py` 最后一行：

```python
app.run(debug=True, host='0.0.0.0', port=5001, use_reloader=False)
```

## 快速解决

最简单的方法：**禁用AirPlay Receiver**

1. 系统设置 → 通用 → AirDrop与隔空播放
2. 关闭"隔空播放接收器"

然后重新启动服务器即可。
